Traversal

Node Neighborhood

Fetches the nodes directly connected to one node, along with the edge that connects them. Use this when you want the immediate local neighborhood only.

Example request: node_id=`6`, direction=`both`, edge_type=`supports`, limit=`20`

Edge type options: `similar_to`, `expands`, `contradicts`, `supports`, `led_to`, `belongs_to_topic`, `derived_from`, `mentions`, `inspired_by`, `part_of`, `reply_to` 


          


          

            
Subgraph Fetch

            

              Traverses outward from one root node and returns the connected subgraph up to the chosen depth and limit.
              Use this to inspect a local cluster around a node.
            

            

              Example request: root node_id=`6`, depth=`2`, limit=`12`, edge_type=`supports`
            

            

              Edge type options: `similar_to`, `expands`, `contradicts`, `supports`,
              `led_to`, `belongs_to_topic`, `derived_from`, `mentions`, `inspired_by`, `part_of`, `reply_to`
            

            

              
              
              
              
              
            

            

          


          

            
Weighted Traversal

            

              Uses the same subgraph traversal, but practically you use it to inspect stronger paths first.
              Returned nodes include `path_score`, which favors paths with stronger edge weights.
            

            

              Example request: root node_id=`6`, depth=`3`, limit=`20`, edge_type=`similar_to`
            

            

              Edge type options: `similar_to`, `expands`, `contradicts`, `supports`,
              `led_to`, `belongs_to_topic`, `derived_from`, `mentions`, `inspired_by`, `part_of`, `reply_to`
            

            

              
              
              
              
              
            

            

          


          

            
Topic Traversal

            

              Starts from a topic node and returns the connected notes around that topic.
              Use this when the root node is a `topic` and you want the topic-centered cluster.
            

            

              Example request: topic node_id=`12`, depth=`2`, limit=`20`
            

            

              
              
              
              
            

            

          


          

            
Outline Planning

            

              Builds a sectioned outline from the traversed subgraph. It groups connected nodes by relation type
              and returns a structured outline for article planning.
            

            

              Example request: root_node_id=`6`, depth=`2`, max_nodes=`12`, edge_type=`supports`
            

            

              Edge type options: `similar_to`, `expands`, `contradicts`, `supports`,
              `led_to`, `belongs_to_topic`, `derived_from`, `mentions`, `inspired_by`, `part_of`, `reply_to`
            

            

              
              
              
              
              
            

            

          

        

      


      

        
Edge Creation

        

          

            
Tag Matcher

            

              Independent matcher that compares internal LLM-generated keywords and concepts. User tags remain separate.
              Tag generation now uses your local Ollama model and edge strength comes from shared generated tags.
            

            

              Example request: function_name=`tag_matcher`, threshold=`0.40`, max_pairs=`20`, extra.model_name=`qwen2.5:3b-instruct`
            

            

              Edge types: `similar_to`, `supports`, `contradicts`, `expands`, `led_to`, `belongs_to_topic`
            

            

              
              
              
              
              
              
              
              
            

            

          


          

            
Embedding Matcher

            

              Independent matcher that scores node pairs using cosine similarity over embeddings.
              Use a global threshold for now.
            

            

              Example request: function_name=`embedding_matcher`, threshold=`0.72`, max_pairs=`50`
            

            

              Edge types: `similar_to`, `supports`, `contradicts`, `expands`, `led_to`, `belongs_to_topic`
            

            

              
              
              
              
              
              
              
              
            

            

          


          

            
LLM Matcher

            

              Independent matcher that asks an LLM to score pairs and choose a basic edge type.
              Run it on all candidate pairs if you want the model to decide directly.
            

            

              Example request: function_name=`llm_matcher`, threshold=`0.60`, max_pairs=`100`
            

            

              Edge types: `similar_to`, `supports`, `contradicts`, `expands`, `led_to`, `belongs_to_topic`
            

            

              
              
              
              
              
              
              
              
            

            

          


          

            
LLM Debator

            

              Independent debate-style matcher with a model arguing for the edge, a model arguing against it,
              and a judge deciding the result. This is a standalone function and not wired to the graph yet.
            

            

              Example request: function_name=`llm_debator`, threshold=`0.55`, max_pairs=`20`
            

            

              Edge types: `similar_to`, `supports`, `contradicts`, `expands`, `led_to`, `belongs_to_topic`
            

            

              
              
              
              
              
              
              
              
            

            

          


          

            
Hub Matcher

            

              Independent hub builder that creates higher-level meaning hubs and links notes upward to them.
              One node can belong to multiple hubs.
            

            

              Example request: function_name=`hub_matcher`, threshold=`0.50`, max_pairs=`30`
            

            

              Edge types: `similar_to`, `supports`, `contradicts`, `expands`, `led_to`, `belongs_to_topic`
